\chapter{Reference Guide}
\label{toc:reference}

This chapter provides a detailed description of the side conditions generated by
the \Region plug-in, via its command-line or via its programmatic
interface, together with the associated optimizations and relevant command-line
options. It is organized by source of side-conditions: Section~\ref{toc:code}
for expression, l-values and \C instructions, Section~\ref{toc:logic} for logic
and \acsl and Section~\ref{toc:functions} for function pre-conditions.
Special cases modify the generated side-conditions as follows:
\begin{description}
  \item[skipped:] the annotation is not generated since it is statically verified;
  \item[invalid:] the annotation is assigned a False-if-reachable status;
\end{description}

\index{region@\verb+region+}
\index{invalid@\verb+invalid+}
Annotations generated through the command-line are tagged with \verb"region"
name. Invalid annotations are tagged with an additional \verb"invalid" property
name. Each category of annotations has also dedicated property names.

\section{Code Conditions}
\label{toc:code}
%% --------------------------------------------------------------------------

Side-conditions are given by alphabetical order or property names with short,
partial although typical examples. Notice that side-conditions associated with
kernel alarms are not generated when they are already computed by another
plug-in, typically \rte or \Eva. Conversely, once annotations
have been emitted by \Region, the associated kernel alarms are set to be
computed, hence further plug-ins may not emit them again.

%% --------------------------------------------------------------------------
\subsection{Aligned}
\index{aligned@\verb+aligned+}

\begin{lstlisting}
//@ check region: aligned: \aligned(p, sizeof(int));
return *p;
\end{lstlisting}

Generated when reading or writing an l-value. Options and optimizations:
\begin{itemize}
  \item option \verb+-no-warn-aligned-pointer+ (skipped);
  \item statically computed modulus of offset (skipped);
\end{itemize}

%% --------------------------------------------------------------------------
\subsection{Bounds}
\index{bounds@\verb+bounds+}

\begin{lstlisting}
//@ check region: bounds: 0 <= k < N;
return a[k];
\end{lstlisting}

Generated when accessing an array element. Options:
\begin{itemize}
  \item access to array element with option \verb+-unsafe-arrays+ (skipped);
\end{itemize}

%% --------------------------------------------------------------------------
\subsection{Initialized}
\index{initialized@\verb+initialized+}

\begin{lstlisting}
//@ check region: initialized: \initialized(p);
return *p;
\end{lstlisting}

Generated when reading an l-value.
Optimizations:
\begin{itemize}
  \item access to global or initialized local variable (skipped);
  \item access to formal parameter of primitive type (skipped);
  \item access to formal parameter of compound type without \verb+\garbage+
  property (skipped);
  \item access to \plugin{CIL}-temporary variable (skipped);
\end{itemize}

%% --------------------------------------------------------------------------
\subsection{Memory Access}
\index{memaccess@\verb+mem_access+}

\begin{lstlisting}
//@ check region: mem_access: \valid(p);
//@ check region: mem_access: \valid_read(q);
*p = *q;
\end{lstlisting}

Generated when accessing an l-value for reading or writing.
Optimizations:
\begin{itemize}
  \item access to a variable within its life-time (skipped);
  \item access to an array with \verb+-safe-arrays+ (skipped);
  \item write-access to read-only region (invalid);
\end{itemize}

%% --------------------------------------------------------------------------
\subsection{Pointer Value}
\index{pointervalue@\verb+pointer_value+}

\begin{lstlisting}
//@ check region: pointer_value: p == \null || \object_pointer(p);
return *p;
\end{lstlisting}

Generated when assigning a pointer value. Options and optimizations:
\begin{itemize}
  \item option \verb+-no-warn-invalid-pointer+ (skipped);
  \item address of array element with \verb+-safe-arrays+ (skipped);
\end{itemize}

%% --------------------------------------------------------------------------
\subsection{Valid Region}
\index{path@\verb+path+, \verb+\valid_region+}

\begin{lstlisting}
//@ check region: path: \valid_region(p+k, sizeof(int));
return *(p+k);
\end{lstlisting}

Generated when accessing a region with pointer arithmetic.
Optimizations:
\begin{itemize}
  \item access to array element with option \verb+-safe-arrays+ (skipped);
\end{itemize}

\section{Logic Conditions}
\label{toc:logic}
Side conditions for logic terms and l-values are not generated by default,
unless option command-line option \verb+-region-logic+ is positioned. They are
similar to side-conditions generated for accessing l-values from code
instructions.

Special \acsl constructions modify the generated side-conditions to
account for typical patterns of conditional access. From a given term or
predicate expression, all memory access through (term) l-values generate the
same side-conditions as for \C code instructions. However, for l-values that
appear under specific patterns, the associated side-conditions are also put
under a specific context.

For instance, considering pattern \verb+A && B+, side-conditions for \verb+A+
are collected as they are, but side-conditions for \verb+B+ are contextualized
by assuming that \verb+A+ is true. Hence, if we consider memory-access only, the
side-condition for predicate \verb$(k > 0 && p[k] == 1)$
is \verb$k > 0 ==> \valid(a+k)$.
Notice that order of evaluation in logical expressions matters for
side-conditions, whereas it does not matter for semantic evaluation.

The patterns taken into account and the associated contexts when generating
side-conditions are listed in Figure~\ref{fig:logic-context}.

\begin{figure}[htbp]
  \begin{center}
    \begin{tabular}{|c|l|}
    \hline \textbf{Pattern} & \textbf{Side-Condition Context} (for \verb+_+) \\
    \hline \verb+A ==> _+ & \verb+ A ==> _+ \\
    \hline \verb+A && _+ & \verb+ A ==> _+ \\
    \hline \verb+A || _+ & \verb+!A ==> _+ \\
    \hline \verb+A ? _ : B+ & \verb+(A ? \true : \false) ==> _+ \\
    \hline \verb+A ? B : _+ & \verb+(A ? \false : \true) ==> _+ \\
    \hline \verb+\let <def>; _+ & \verb+\let <def>; _+ \\
    \hline \verb+\forall <decl>; _+ & \verb+\forall <decl>; _+ \\
    \hline \verb+\exists <decl>; _+ & \verb+\forall <decl>; _+ \\
    \hline \verb+\lambda <decl>; _+ & \verb+\forall <decl>; _+ \\
    \hline \verb+{ E | <decl>; _ }+ & \verb+\forall <decl>; _+ \\
    \hline \verb+{ _ | <decl>; P }+ & \verb+\forall <decl>; P ==> _+ \\
    \hline
    \end{tabular}
  \end{center}
  \caption{Contexts for Logical Side-Conditions}
  \label{fig:logic-context}
\end{figure}
